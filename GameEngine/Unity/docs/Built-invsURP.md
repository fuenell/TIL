# Built-in vs URP

# Built-in (포워드)
유니티의 레거시 파이프 라인은
Light의 개수 * 물체의 개수 만큼의 드로우 콜이 발생한다?
(진짜인가?)
대신 라이트 수 제한이 없다

멀티 패스 방식

# URP (포워드)
URP는 Light 수 제한이 8개
하지만 한번의 드로우콜로 그린다 (포워드 렌더링 기준)

싱글 패스 방식

# 멀티 패스 방식이 존재한 이유
Built-in 방식이 여러모로 단점만 있는 것 같지만
과거에는 상수 버퍼의 크기가 작았고, 동적 분기의 성능이 형편없었다. 그래서 라이트가 추가될 때마다 파이프라인을 쪼개서 단순한 연산을 여러번 반복하는 것이 당시 하드웨어 환경에서는 최선이었지만.
하드웨어가 발전한 현재는 그 실효성이 없어졌다.

따라서 무조건 Built-in 보다 URP가 더 좋다.
